The loops are no good!

So, the program does find loops for you but they aren't seamless (have clicks and pops at the loop point). Here are some strategies to (if possible) improve the results. This is especially useful strategies if the program really returns as many loops as you ask for.

  • Increase number of loops to return (this will give you more alternatives to choose between)
  • Increase the loop pool multiple (will give the program more loops to choose between)
  • Use the sliders to manually adjust start and end position for the sustainsection
  • Increase number of candidates
  • Shorten/change the minimum distance between loops (at 0 all candidates will be tested)
  • Play with the minimum loop length (see above though!)
  • Decrease (play with) the quality value, which tell the program to only accept more correlated loop points
  • Test searching with brute force enabled
  • Start over again at the top of the list and adjust the values again if possible...

If you find that not as many loops are returned as you ask for and still the quality of those found are not good enough, you might have to combine the changes from the No loops section with these until you get something good. So far this autosearch has produced at least as good loops as any other software I've tried, but the settings really have to be tweaked quite a bit in certian cases.

As a last resort you can always try crossfading the loops and see if that helps.
