Credits

LoopAuditioneer uses a custom version of libsndfile for file I/O handling that contain a few modifications by Lars Palo mainly to allow for embedding/reading correct MIDIPitchFraction numbers. The modified source is available in the lib-src folder on sourceforge accessible by SVN.

For sound the RtAudio library is used.

If samplerate conversion is necessary for audio output the libsamplerate library is used. However, the main reason for this to be necessary is if you're using Jack on Linux as audio Api. It would most likely be better to change the samplerate that is used by Jack to match the one of the actually loaded sample file to be sure that optimal fidelity of audio is preserved - and by that giving you the best possibilities of correctly auditioneering the loops and cues in the file.

The FFT routines is taken from the Audacity source and (FFT.cpp and FFT.h) has been slightly modified to better be integrated into LoopAuditioneer.

The algorithm for pitch detection in the time domain is taken and adapted from a discussion on the GrandOrgue mailing list.
